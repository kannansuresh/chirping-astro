Cet article est un petit **essai photo** qui illustre les
fonctionnalités d'image du thème de bout en bout. Notez ce que cette
page fait que les autres ne font pas :

- Le hero est une **image locale** issue de
  `public/images/coastline.jpg`, pas une URL CDN distante.
- `toc: false` — aucune table des matières à droite. Les essais photo
  se lisent mieux sans.
- `showFeaturedImage: true` est défini explicitement, même si c'est
  le défaut du site. Les auteurs peuvent désactiver par article.

## Où mettre les images

Vous avez trois options de stockage :

### 1. Chemin public (le plus simple)

Déposez le fichier dans `public/images/` (ou n'importe où sous
`public/`) et référencez-le par chemin absolu :

```yaml
heroImage: /images/coastline.jpg
heroImageAlt: Vagues sur une côte rocheuse en longue exposition
```

Pour : zéro cérémonial, facile à remplacer. Contre : pas
d'optimisation automatique (Astro ne traite rien dans `public/`).

### 2. Asset importé (en MDX)

Pour bénéficier du pipeline d'image d'Astro — redimensionnement
automatique, formats modernes, dimensions injectées — importez
l'asset dans un fichier MDX et utilisez `<Image>` :

```mdx title="src/content/posts/fr/exemple.mdx"
import { Image } from 'astro:assets';
import coastline from '../../../assets/coastline.jpg';

<Image src={coastline} alt="Une côte au crépuscule" widths={[400, 800, 1600]} />
```

Notez que `heroImage` dans le frontmatter est une chaîne et ne
supporte pas les assets importés — pour le **hero**, utilisez chemins
publics ou URL externes. Pour les images en ligne, MDX + `<Image>`
est meilleur quand vous voulez une sortie optimisée.

### 3. URL externe

Les URL distantes fonctionnent. Utile pour Unsplash, votre CDN, ou un
ancien hôte :

```yaml
heroImage: https://images.unsplash.com/photo-1499951360447-b19be8fe80f5?w=1600&q=80&auto=format&fit=crop
heroImageAlt: Lumière douce du jour sur un bureau en bois
```

Pour : pas de gonflement du commit. Contre : dépendance externe ;
l'image disparaît si l'hôte disparaît.

## Règles d'image hero

Le hero apparaît à deux endroits :

1. Comme **vignette de carte** (page d'accueil, listings, archives,
   pages de tag et de catégorie).
2. Comme **grand hero** sur la page de l'article.

Les deux sont contrôlés par :

- `SITE.showFeaturedImages` dans
  [`src/config.ts`](src/config.ts) — défaut global.
- `showFeaturedImage` par article — override le défaut global.
- `heroImage` par article — fournit la source réelle.

Si `heroImage` est absent, aucun hero n'est rendu, drapeaux ou non.
Si `heroImage` est présent mais `showFeaturedImage: false`, la page
de l'article est sans image — mais la carte sur les listings peut
toujours utiliser le hero, selon votre personnalisation de mise en
page.

## Images en ligne dans la prose

Les images Markdown en ligne occupent toute la largeur de la prose :

![Roches de côte photographiées sous un ciel couvert](https://images.unsplash.com/photo-1518837695005-2083093ee35b?w=1600&q=80&auto=format&fit=crop)

Une figure légendée avec `<figcaption>` :

<figure>
  <img
    src="https://images.unsplash.com/photo-1505142468610-359e7d316be0?w=1600&q=80&auto=format&fit=crop"
    alt="Le soleil perçant les nuages au-dessus de l'océan"
    loading="lazy"
  />
  <figcaption>Soleil sur l'horizon. Photo de Sean O. sur Unsplash.</figcaption>
</figure>

Images côte à côte ? Mettez-les dans un `<div>` HTML avec des classes
Tailwind (c'est du MDX, après tout) :

<div class="grid gap-3 sm:grid-cols-2">
  <img
    src="https://images.unsplash.com/photo-1507525428034-b723cf961d3e?w=900&q=80&auto=format&fit=crop"
    alt="Une plage de sable"
    loading="lazy"
    class="rounded-lg"
  />
  <img
    src="https://images.unsplash.com/photo-1519046904884-53103b34b206?w=900&q=80&auto=format&fit=crop"
    alt="Une falaise côtière"
    loading="lazy"
    class="rounded-lg"
  />
</div>

## Texte alternatif — écrivez-en

`heroImageAlt` est techniquement optionnel. Ne le traitez pas ainsi.

- **Images décoratives** (un bandeau coloré en haut d'un article) :
  alt vide — `heroImageAlt: ""`.
- **Images de contenu** (capture d'écran, photo, graphique) :
  décrivez ce que l'image transmet, pas le fichier.
- **Ne commencez pas par « Image de… »** — les lecteurs d'écran
  l'annoncent déjà comme image.

Le thème utilise `heroImageAlt` pour le hero **et** la vignette de
carte, donc un alt bien écrit sert les deux contextes.

## Astuces de performance

- Les images en ligne reçoivent `loading="lazy"` automatiquement
  quand écrites en HTML brut, mais la syntaxe Markdown `![]()` non.
  Préférez `<img loading="lazy">` pour les images en bas de page.
- Les hero apparaissent au-dessus de la ligne de flottaison —
  réglez-les sur `loading="eager"` via le layout si vous avez un
  hero critique à prioriser.
- Utilisez le composant `<Image>` d'Astro pour ce que vous contrôlez
  vous-même (assets locaux) afin d'obtenir formats modernes et
  sortie multi-largeurs.

<Callout type="info" title="Astuce">
  Mettez <code>showFeaturedImage: false</code> sur les articles très textuels où le hero n'apporte
  rien — le lecteur entre directement dans les mots.
</Callout>

Cet article fait office de référence exhaustive pour chaque élément textuel
rendu par le thème. Il sert à vérifier que le CSS, le choix des polices et
les blocs de code restent cohérents en mode **clair** _et_ **sombre**.

## Paragraphes et formatage en ligne

Astro prend en charge toute la grammaire Markdown standard. Vous pouvez
écrire en **gras**, en _italique_, ~~barré~~, en `code en ligne`, voire
mêler `**du gras dans du code**` (rendu en code brut). Les combinaisons
**_gras et italique_** ou `code avec **balisage**` fonctionnent aussi.
Les longues URLs deviennent cliquables : [astro.build](https://astro.build).

> « La typographie est l'art de donner au langage humain une forme
> visuelle durable. » — _Robert Bringhurst_

Un second paragraphe maintient le rythme. La hauteur de ligne, les marges
et la `max-width` sont calibrées pour une lecture confortable sur écran
comme sur mobile.

### Un sous-titre H3

Sert à découper les longues sections. Le sommaire de droite n'affichera
ces H3 que lorsque vous serez à l'intérieur de cette section H2.

#### Et un sous-titre H4

Pour les contenus très denses. Rarement nécessaire, toujours pris en charge.

## Listes

Liste non ordonnée :

- Premier élément avec un [lien vers l'accueil](/fr/).
- Deuxième élément avec du **gras** et du `code en ligne`.
- Élément imbriqué :
  - Avec ses propres enfants.
  - Et un sous-élément encore plus profond.
    - Trois niveaux d'imbrication, sans souci.

Liste ordonnée :

1. Choisissez un sujet.
2. Construisez le plan — H2 pour les sections, H3 pour les sous-sections.
3. Rédigez d'un trait, sans corriger.
4. Laissez reposer une nuit.
5. Éditez sans pitié.

Liste de tâches (GFM) :

## Code en ligne et raccourcis clavier

Utilisez le `code en ligne` pour mettre en avant un identifiant
(`SITE.showFeaturedImages`) ou un court extrait (`bun astro dev`). Pour
les raccourcis clavier, préférez la balise `<kbd>` : appuyez sur
<kbd>Cmd</kbd> + <kbd>K</kbd> pour ouvrir la recherche.

## Blocs de code

Exemple TypeScript :

```ts
// utils/posts.ts
import { getCollection, type CollectionEntry } from 'astro:content';
import { SITE, type Locale } from '../config';

export function shouldShowHero(post: CollectionEntry<'posts'>): boolean {
  if (!post.data.heroImage) return false;
  return post.data.showFeaturedImage ?? SITE.showFeaturedImages;
}
```

Exemple bash :

```bash
# Lance le serveur de développement, puis le build de production
bun install
bun run dev
bun run build
```

Un diff :

```diff
- const heroSrc = typeof img === 'string' ? img : img?.src;
+ const heroSrc = heroImageSrc(post);
+ const showHero = shouldShowHero(post);
```

Un frontmatter YAML :

```yaml
title: Bonjour le monde
pubDate: 2026-04-26
tags: [astro, theme]
heroImage: /images/coastline.jpg
showFeaturedImage: true
```

## Encadrés (callouts)

Le composant MDX `<Callout>` propose quatre variantes sémantiques :

<Callout type="info" title="Info">
Le thème daisyUI actif pilote la couleur de chaque encadré, qui reste
lisible en mode clair comme en mode sombre.
</Callout>

<Callout type="success" title="Succès">
Vous venez de publier un article propre. Récompensez-vous.
</Callout>

<Callout type="warning" title="Attention">
Évitez de définir manuellement `lang:` dans le frontmatter sauf nécessité —
le thème déduit la langue du chemin de fichier.

</Callout>

<Callout type="error" title="Erreur">
Oublier `pubDate` provoque une erreur Zod claire au build.
</Callout>

En Markdown classique, vous pouvez aussi utiliser la classe `callout` :

<div class="callout callout-info">
  <strong>Note :</strong> définissez <code>toc: false</code> dans le
  frontmatter pour masquer la table des matières d'un article.
</div>

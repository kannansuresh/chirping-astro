This post is a kitchen-sink reference for every text element the theme

themes all stay coherent in **light** _and_ **dark** mode.

## Paragraphs and inline formatting

Astro fully supports the standard Markdown inline grammar. You can write
**bold**, _italic_, ~~strikethrough~~, `inline code`, and even nested
`**bold inside code**` (rendered as plain code). Combinations like
**_bold italic_** and `code with **markup**` work too. Long URLs
turn into clickable links: [astro.build](https://astro.build).

> "Typography is the craft of endowing human language with a durable
> visual form." — _Robert Bringhurst_

A second paragraph keeps the rhythm. The line height, margin, and
`max-width` are tuned for comfortable reading on both desktop and mobile.

## Inline code and keyboard shortcuts

Reach for `inline code` to highlight identifiers (`SITE.showFeaturedImages`)
or short snippets (`bun astro dev`). For keyboard shortcuts, prefer the
`<kbd>` tag: press <kbd>Cmd</kbd> + <kbd>K</kbd> to open search.

## Fenced code blocks

A TypeScript example:

```ts title="src/utils/posts.ts" ins={5-8}
// utils/posts.ts
import { getCollection, type CollectionEntry } from 'astro:content';
import { SITE, type Locale } from '../config';

export function shouldShowHero(post: CollectionEntry<'posts'>): boolean {
  if (!post.data.heroImage) return false;
  return post.data.showFeaturedImage ?? SITE.showFeaturedImages;
}
```

A bash example with a frame title:

```bash title="Terminal" frame="terminal"
# Run the dev server, then build for production
bun install
bun run dev
bun run build
```

A diff with a label:

```diff title="Refactor: hero image source"
- const heroSrc = typeof img === 'string' ? img : img?.src;
+ const heroSrc = heroImageSrc(post);
+ const showHero = shouldShowHero(post);
```

```yaml

```

## Callouts

The `<Callout>` MDX component renders one of four semantic variants:

<Callout type="info" title="Info">
  The active daisyUI theme drives every callout colour, so they stay readable in both light and dark
  mode.
</Callout>

<Callout type="success" title="Success">
  You just shipped a clean post. Treat yourself.
</Callout>

<Callout type="warning" title="Warning">
  Avoid manually setting `lang:` in frontmatter unless you really need to — the theme infers it from
  the file path.
</Callout>

<Callout type="error" title="Error">
  Forgetting `pubDate` will cause the build to fail with a clear Zod error.
</Callout>

In plain Markdown you can also reach for the lower-level callout class:

<div class="callout callout-info">
  <strong>Note:</strong> set <code>toc: false</code> in frontmatter to hide the table of contents on
  a per-post basis.
</div>
